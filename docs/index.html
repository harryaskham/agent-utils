Publishing and preview

GitHub Pages path: Repository Settings → Pages → Deploy from branch: main, folder: /docs

  • animate a loaded series

App automation Pi extension

app_automation_list, app_automation_doctor, app_automation_overview, app_automation_plan, app_automation_run, app_automation_open_bundle_run_once, app_automation_refresh_start, app_automation_refresh_bundle_start, app_automation_refresh_bundle_run_once, app_automation_refresh_staleness, app_automation_refresh_stale_run_once, app_automation_refresh_status, app_automation_refresh_stop, app_automation_snapshots_list, app_automation_snapshots_digest, app_automation_snapshot_links, app_automation_snapshots_staleness, app_automation_snapshots_cleanup_plan, app_automation_snapshot_read, app_automation_status, /tendril-app

extensions/app-automation.js, extensions/app-automation/catalog.js, docs/app-automation.md

  • optionally probe Tendril target discovery from app_automation_doctor
  • summarize work-app refreshers and snapshots
  • plan Slack notification snapshot
  • dry-run or execute allowlisted runner steps
  • plan Markdown-to-canvas sync
  • inspect canonical app automation state root
  • list/digest/query timestamped snapshot links with freshness/app/kind counts, source context, explicit kind filtering, and clear empty-filter output
  • list/freshness-check/cleanup-plan/read persisted app snapshots
  • use flexible /tendril-app links filters such as all fresh kind:events.snapshot standup
  • start/inspect/stop periodic app refreshers
  • open Slack/Calendar/Outlook/Teams browser surfaces once
  • arm, run once, inspect per-action staleness, or stale-refresh the standard Slack/Calendar/Outlook/Teams refresh bundle
